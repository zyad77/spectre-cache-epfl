⚡ WSL2 · Ubuntu · Intel Core i7 · gcc -O2

Environnement WSL2

Environnement de départ du projet. La virtualisation de rdtsc par Hyper-V introduit des biais de mesure qui ont nécessité plusieurs ajustements — notamment la calibration du threshold et le passage à un PoC alternatif pour démontrer la fuite spéculative.

HISTORIQUE Problèmes rencontrés & corrections
PROB Faux positif systématique index 0. Avec FLUSH_THRESHOLD=80 (valeur initiale), le programme retournait toujours l'index 0, quelle que soit la valeur secrète. Taux brut : 0/8.
FIX Calibration à 200 cycles. Mesure réelle des latences (hit=42 cycles, miss=246 cycles) via calibrate.c. Threshold optimal calculé entre les deux. Résultat : 8/8 en guess brut, sans filtrage.
PROB Spectre V1 classique KO. 0/15 bytes — le programme retourne du bruit (0x00, 0xCA, 0xC8). Mitigations kernel + virtualisation rdtsc masquent le signal spéculatif.
CHANGE Passage au PoC alternatif (spectre_poc.c). Flush de condition au lieu de array1_size, probe array 256 pages × 4 KiB, mesure par minimum de latence sur 100 rounds, vote majoritaire sur 12 répétitions.
RÉSULTAT Fuite confirmée sous WSL. Mode [Slow] reconstruit "TestStringLeakedBySpectre" complet sur plusieurs runs. Fence Validator confirme réduction 99% avec LFENCE.

Linux sans mitigations

Même machine (i5-8350U, noyau 6.17.0-29-generic) avec mitigations=off au boot GRUB, prefetcher désactivé (wrmsr -a 0x1a4 0xf), et governor CPU en performance. Configuration de laboratoire maximale — permet de démontrer la fuite spéculative réelle et l'efficacité de la fence.

HISTORIQUEProblèmes rencontrés & corrections
PROB Flush+Reload V1 toujours saturé, même sans mitigations. Tous les 256 index retournent score=1000. Spectre V1 donne 0/14 même avec mitigations=off.
ANALYSE Cause : clflushopt asynchrone. Le CPU i5-8350U utilise clflushopt (version asynchrone de clflush). Le prefetcher recharge les lignes avant la mesure. Ce n'est pas les mitigations — c'est une limitation hardware spécifique à ce CPU.
TENTATIVE STRIDE 4096, randomisation, prefetcher off via wrmsr. Aucune de ces modifications ne résout la saturation du canal V1. Le problème est dans l'implémentation du flush, pas dans l'environnement.
FIX PoC alternatif (spectre_v2_test.c). Probe array 256 pages × 4 KiB (1 MB total), mesure par minimum de latence, vote majoritaire 12 répétitions. Contourne le problème clflushopt.
RÉSULTAT Spectre fonctionne : 20/25 chars en meilleure run. Fence Validator : réduction 94%. Démonstration complète sans/avec fence.

Conclusion générale : le canal auxiliaire par cache existe et est mesurable dans les trois environnements.
