Problématique

La question centrale

Research Question
Comment l'exécution spéculative, conçue pour améliorer les performances des processeurs, peut-elle être exploitée pour provoquer une fuite d'informations sensibles via le cache, malgré l'existence de mécanismes de protection tels que les barrières de spéculation ?
⚡

Exécution spéculative

Les processeurs modernes prédisent les branchements et exécutent du code en avance. Si la prédiction est fausse, les effets sont annulés — mais le cache garde des traces.

🔍

Canal auxiliaire (cache)

Les temps d'accès mémoire révèlent si une adresse est en cache ou non. Un attaquant peut mesurer ces différences (~4ns vs ~100ns) pour inférer des données secrètes.

🛡️

Barrières de spéculation

Des instructions comme LFENCE ont été introduites pour bloquer la spéculation. Mais leurs effets réels varient selon les architectures matérielles.

Objectifs

Ce que ce projet explore

Six axes d'étude, du mécanisme fondamental jusqu'aux limites des protections actuelles.

01

Fonctionnement de Spectre

Comprendre le rôle de la prédiction de branchement, de l'exécution spéculative et du cache dans la fuite d'informations.

02

Rôle de la prédiction de branchement

Montrer comment une optimisation matérielle conçue pour la performance peut devenir une faiblesse exploitable.

03

Cache comme canal auxiliaire

Illustrer comment les traces laissées dans le cache après une exécution spéculative annulée peuvent être observées via des mesures de temps d'accès.

04

Conséquences en sécurité

Analyser les implications pour la confidentialité des données dans les systèmes modernes — données normalement inaccessibles déduites sans contourner les protections classiques.

05

Fonctionnement des barrières

Expliquer le principe général des barrières de spéculation (LFENCE, MFENCE) et comment elles sont censées réduire le risque de fuite.

06

Limites des barrières

Étudier les conditions dans lesquelles ces protections peuvent montrer des faiblesses ou laisser subsister des comportements exploitables.
